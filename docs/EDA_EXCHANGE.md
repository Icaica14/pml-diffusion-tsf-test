# EDA — Exchange (the iteration dataset)

*A short, figure-backed synthesis of the exploratory analysis in
[`notebooks/01_eda_exchange.ipynb`](../notebooks/01_eda_exchange.ipynb). Every number
below is regenerated by that notebook into
[`results/eda_exchange_stats.json`](../results/eda_exchange_stats.json); every figure
is saved to [`figures/`](../figures). Nothing here is typed by hand.*

> **Why Exchange, and why now.** Exchange is the **iteration** dataset (plan §4.3):
> small and fast, so we bring the whole pipeline green here *before* scaling to the
> primary dataset (Solar/Electricity, a team decision still open). The EDA runs on the
> **same data contract every model consumes** (`src/data/exchange.py`), so what we see
> here is exactly what the models will see.

## The dataset at a glance

| | |
|---|---|
| Series | **8** currencies vs USD (`D = 8`) |
| Cadence | daily (nominal index; the raw file has no timestamps) |
| Length | **7 588** steps → train **5 311** / val **758** / test **1 519** (70 / 10 / 20, time-ordered) |
| Context / horizon | `H = 60`, `τ = 30` (defaults in `configs/data_exchange.yaml`) |
| Source | LSTNet multivariate benchmark (the lineage TimeGrad uses) |

The split is **contiguous and time-ordered** (test = the most recent slice); the scaler
is **fit on train only**. Both are enforced by the data contract and pinned by the
offline tests in `tests/test_data_contract.py`.

## Six findings (and what each one means for the project)

**1. Levels behave like a random walk (near unit root).**
ADF fails to reject non-stationarity in **7 of 8** series (median p ≈ **0.41**), and the
lag-1 autocorrelation of levels is **0.999**. See `figures/eda_exchange_series.png` and
`figures/eda_exchange_acf.png`.
→ A **naive "tomorrow = today" forecast (M0) is a genuinely hard baseline**; beating it
must be *earned*. This is exactly the honesty anchor the plan wants (§5.1).

**2. Log-returns are stationary and ~white in the mean.**
ADF rejects a unit root for every return series (max p ≈ **0.0**); the lag-1 ACF of
returns is only **−0.10** and decays to noise immediately.
→ Little *linear* signal in the direction of moves, so ARIMA/ETS (M1) gains will be
modest — which is precisely why richer probabilistic models are worth testing.

**3. Volatility clusters (strong heteroskedasticity).**
Squared returns are positively autocorrelated (lag-1 ≈ **0.42**); volatility comes in
bursts (`figures/eda_exchange_volatility.png`).
→ A good forecaster must produce **state-dependent interval width** — wider in
turbulent regimes, tighter in calm ones. This is where calibrated probabilistic models
(scored by CRPS / coverage) should separate from point forecasts.

**4. Returns are heavy-tailed.**
Pooled excess kurtosis ≈ **601**, skew ≈ **−3.6** (`figures/eda_exchange_returns_dist.png`),
driven by a few extreme single-day jumps (a likely re-peg / data break in one currency).
→ A **Gaussian likelihood (DeepAR-Gaussian) will under-fit**; Student-t or a flexible
generative model (diffusion) is better motivated. The jumps also feed the **E4** story.

**5. No calendar seasonality.**
ACF of returns at business-week (lag 5) and monthly (lag 21) horizons is ≈ 0.
→ Deliberate contrast: **Exchange tests the trend / volatility-regime axis**, while the
*primary* dataset (Solar/Electricity) will test the **strong-seasonality** axis. Using
both stops us from over-fitting our conclusions to one kind of structure.

**6. Volatility regimes exist → E4 material.**
Rolling 60-day volatility alternates long calm stretches with turbulent bursts
(`figures/eda_exchange_regimes.png`).
→ For **E4** we evaluate on a window containing a turbulent burst and check whether the
probabilistic models *widen* their intervals appropriately (coverage holds) while point
models degrade.

## Implications for `H` / `τ`

With context `H = 60` (≈ a quarter of trading days) the model sees a full recent
trend-and-volatility regime; horizon `τ = 30` (≈ 6 trading weeks) is long enough that
the random-walk levels can drift materially — so **calibrated uncertainty, not point
accuracy, is what should separate the models**. These defaults live in
`configs/data_exchange.yaml`; the horizon sweep **E2** stress-tests them.

## Caveats

- **Timestamps are nominal.** The raw file ships without dates; we attach a daily index
  from 1990-01-01 for plotting/covariates only. We never read real-world events off
  absolute dates — the index is just "trading day *t*".
- **Findings are dataset-specific.** They justify Exchange's role as the fast iteration
  dataset; the headline scientific claims of the project are made on the primary dataset
  once the pipeline is green here.
