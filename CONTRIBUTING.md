# Contributing

This is a 3-person exam project. The goal is a clean, reproducible comparison study, not a sprawling codebase. Read [`docs/IMPLEMENTATION_PLAN.md`](docs/IMPLEMENTATION_PLAN.md) (or the Italian [`docs/IMPLEMENTATION_PLAN_IT.md`](docs/IMPLEMENTATION_PLAN_IT.md)) before writing code — especially Part 8 (engineering) and Part 9 (who owns what).

## The golden rule

> We are not presenting a survey, and we are not building a black box. One well-defined probabilistic question, answered with real experiments, every model openable and explainable in the course's own notation.

## Roles (own a vertical slice end-to-end)

- **A — Baselines & Statistics:** M0, M1, point metrics, classical-forecasting narrative.
- **B — Diffusion & Infrastructure:** M2 (DeepAR), M3 (TimeGrad), the toy DDPM, Colab/training, configs.
- **C — Evaluation, Viz & Story:** CRPS/coverage/calibration, all figures, the slide narrative, the oral Q&A bank.

Add your name + GitHub handle to the table in the README.

## Workflow

1. **Branch per task:** `feature/<area>-<short-desc>` (e.g. `feature/data-exchange-loader`, `feature/eval-crps`). Never commit straight to `main`.
2. **One config = one run.** No magic numbers in code — everything (model, data, `H`, `τ`, `T`, seed) goes in a YAML under `configs/`.
3. **The results registry is the truth.** Every experiment appends a row to `results/registry.csv`; tables and figures are *generated* from it, never typed by hand.
4. **Open a Pull Request** and have one teammate review it against the checklist below.

## Code-review checklist (before merge)

- [ ] No scaling/leakage across train/val/test splits
- [ ] Same `H`, `τ`, and test windows across all models
- [ ] Seed set and logged
- [ ] Metric matches its definition (state CRPS-sum vs mean)
- [ ] Every figure is regenerable from a script
- [ ] Config committed alongside the result

## Definition of "done" for an experiment

An experiment is done when it has: a stated hypothesis, a fixed setup, the measured numbers in `results/registry.csv` (mean ± std over ≥3 seeds for learned models), and an artifact (table or figure) generated by a script. See Appendix E of the plan for the full methodology + runbook.

## What never goes in this repo

Course notes, textbooks, or any copyrighted PDF (the professor's notes must not be redistributed). Raw datasets and training checkpoints are gitignored — link to them or document how to download instead.
